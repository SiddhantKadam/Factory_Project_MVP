Cutting — Material Issue & Profile Cutting

Owner: Project Manager · Machine operator executes. The traceability spine of the whole job starts here.

Why these elements — real-world mapping
  • Cut List table — the operator works from a part-by-part list (mark, profile, length, qty, method), not a generic checklist. Per-row status shows shop progress.
  • Material traceability — steel jobs are audited on heat number → MTC. If this isn't captured at cutting, the whole job fails traceability at QC/dispatch.

  • Hard-stamp confirmation — every piece must carry its part mark + heat no. downstream; it's a required checklist gate.
Phase 02 / 05 · Beamline

Real shop flow: Cut beam loaded to CNC line (e.g. Peddinghaus) → drill bolt holes per connection detail → cope / notch beam ends for framing → mark fitting layout (cleats, stiffeners) → verify hole positions & part mark.

Why these elements — real-world mapping
  • Per-member operations table — beamline work is tracked member-by-member across 3 operations (drill / cope / mark). A single % bar hides which members are stuck.

Phase 03 / 05 · Fit-Up

Fit-Up — Assembly, Tack & HOLD Inspection

Owner: Quality Inspector · This is a HOLD point — full welding cannot start until fit-up is passed.

Real shop flow: Fitter assembles main member + fittings per layout → sets root gap & alignment → tack welds → inspector checks dimensions (length, squareness, camber, diagonal) and weld joint prep → Pass = release to weld / Fail = rework / Dispute = escalate.

Why these elements — real-world mapping
  • HOLD-point banner + 3-way action (Approve / Reject / Dispute) — fit-up is a mandatory inspection gate; welding is literally blocked until pass. This is the single most important control on the screen.
  • Full dimensional survey — real inspectors record length, squareness, camber, diagonal, root gap AND weld-prep (bevel, root face) — not just 2 fields. Each shows value vs tolerance.
  • Fitting BOM — fit-up = assembling many small fittings onto the member; a checklist per fitting mark mirrors the shop traveller.
  • Weld reference panel (WPS, joint type, welder ID) — traceability for the weld that follows.
Phase 04 / 05 · QC

QC — Weld NDT, Dimensional & Paint/DFT

Owner: Quality Head · Often includes Third-Party Inspection (TPI) hold points. Releases the member to dispatch.

Why these elements — real-world mapping
  • NDT log with method + extent + result — QC on welded steel is driven by NDT (UT/MPI/DPI/RT). Each weld's method, coverage % and accept/reject must be recorded for the data book.
  • NCR handling — a rejected weld raises a Non-Conformance Report, gets repaired, and is re-inspected. The UI must show open vs closed NCRs; a job can't dispatch with open NCRs.
  • DFT table with surface-prep grade — painting acceptance = blast standard (Sa 2.5) + DFT per zone vs spec range. Kept from current build, extended with prep column.
  • ITP / TPI sign-off — client or third-party inspector witnesses hold points; "Approve QC" is what unlocks Dispatch.
Phase 05 / 05 · Dispatch

Real shop flow: QC released → reconcile piece count vs BOM → weigh (gross/net) → mark & pack per erection sequence → raise delivery note / e-way bill → load (photos) → transport → capture POD (proof of delivery) signature.

Why these elements — real-world mapping
  • Piece reconciliation vs BOM — the real dispatch gate is "does loaded count = QC-passed count = BOM?". A short-shipment must be caught here, so the "Mark Dispatched" button stays disabled until every mark reconciles.
  • E-way bill + delivery note — statutory for road movement of goods (India context). Both are captured, not just an internal note.
  • Weight & package count — transporter and client both need gross/net weight and bundle count.
  • POD (proof of delivery) — the project only truly closes when the client signs receipt; POD status is tracked to 100%.
Patterns shared across all 5 phases
Consistent shell — what stays the same every phase
  • Role dropdown + access pill in the top bar — the same 8 roles drive edit vs view; owners differ per phase (PM → PM → QI → QH → DI).
  • 5-step stepper with lock icons on future phases — a phase can't start until its predecessor is Approved (sequential gating).
  • Hero card: phase number, live status, project facts, completion %, and the primary action (Complete / Approve / Dispatch).
  • Left = work (phase-specific tables + checklist + comments), Right = documents + info. Every phase carries traceability docs forward.
  • Required (*) checklist gates — the primary action stays disabled until all required items are checked, mirroring shop-floor sign-off discipline.
  • Comment + Dispute thread on every phase — non-owners (e.g. GM, Planning) can watch, comment, and escalate without editing.
